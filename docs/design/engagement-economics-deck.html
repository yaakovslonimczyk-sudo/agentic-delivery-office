The gap today

The cockpit tracks activity, not cost accountability

Three things the CTO can't see or audit per engagement right now.

Aggregate tokens only

The board shows one rolled-up tokens number per engagement. No phase split, no agent split — no way to see where the spend went.

Gate decisions overwrite

Each transition overwrites engagement state. There is no history[] — so no real timeline, and no audit trail of how a gate was reached.

The savings claim is invisible

The kit's headline economics — junior routing saves ~30–40% — isn't surfaced or auditable per engagement. It's asserted, never shown.

The drill-down

Click a row → a two-section engagement panel

Section 1 · Gate / Phase Timeline

An ordered walk through A → Gate 1 → B → Gate 2 → C → Delivered, rendered from the new history[] of transitions — done / active / pending nodes, each with note and timestamp.

Section 2 · Token → $ Breakdown

By-phase and by-agent tables with a SENIOR / JUNIOR tier split, a headline savings tile, and a reconciliation footer proving Σ agent tokens = board total.

Resolved & defended

Four design decisions · top risks

6.1History granularity

One {phase,status,note,ts} appended per office-state.sh call — matches existing call sites exactly, no extra instrumentation.

6.2Lossless backfill

Seed-on-first-append: old files synthesize a seed entry from current state before appending. Zero migration script, zero downtime, no field lost.

6.3Prices hardcoded now

PRICES = {SENIOR:15, JUNIOR:3} in index.html, labeled illustrative. Editable-in-Settings deferred to the backlog — honors zero-deps / no-build.

6.4"Other / unattributed" bucket

When Σ agent tokens ≠ aggregate, a visible residual row appears rather than silently absorbing the gap. Footer shows ✓ only when unattributed = 0. Honesty by construction.

Top risks & mitigations

R8 — two byte-identical board copies. .claude/board/ and the ~/.claude/office-board/ mirror are identical today; the build must ship to both copies (one-time mirror).  Also: R4 element collision — mandate a separate #eng-side-panel; R1 phase attribution is a heuristic (by-agent is exact); R7 prices clearly footnoted as illustrative, not billing.

Build plan & the ask

MVP slice end-to-end, then harden

MVP slice — dependency order
  1. office-state.sh — append-with-seed history[]. Unblocks real timeline data.
  2. server.js — carry history + new /api/economics scan & bucketing.
  3. index.html data — PRICES, agentTier(), static demoDetailData.
  4. Panel shell — #eng-side-panel + openEngDetail() + per-row click.
  5. Render — timeline + economics with savings stat & reconcile bucket.
Hardening backlog — post-MVP
  • Editable prices in the Settings tier panel (decision 6.3 deferred item).
  • Per-model pricing & a configurable tier→price table.
  • Cost time-series / sparkline; cross-engagement rollup.
  • Memoize the economics scan by file size+mtime if the log grows.
  • Richer timeline (duration deltas); CSV/JSON export for finance.
  • Surface persistent unattributed as a roster-gap signal.
